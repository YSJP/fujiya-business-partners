# Astro 本番切替後の監視項目

## 目的

Astro 本番切替後 1 か月の監視項目を固定し、異常検知とロールバック判断を早くする。

## 公開リポジトリで扱わない情報

- 内部 issue URL
- 非公開スクリーンショット URL
- 担当者の個人連絡先
- 監視用の private dashboard URL

## 監視期間

- 重点監視: 切替直後から 7 日間
- 通常監視: 8 日目から 30 日目
- 旧構成の保持判断: 30 日目終了時に実施

## 重点監視対象

- `/`
- `/faq/`
- `/privacy/`
- `/form/`
- スマホ表示
- 問い合わせ導線

## 毎日確認する項目

- Home のファーストビューが崩れていない
- FAQ の開閉が動作する
- Privacy の本文が読める
- Form から Google Forms へ遷移できる
- ハンバーガーメニューが開閉できる
- 主要リンクに underline がある
- スマホで左右余白が広すぎない
- footer が不自然に浮いていない

## 切替当日に確認する項目

- `fujiya-bp.com` が Astro 本番を表示している
- `www` 有無の導線が意図どおり
- Cloudflare Pages の最新 deploy が成功している
- Home / FAQ / Privacy / Form の HTTP 200 を確認する
- favicon と OG 画像が 404 にならない

## 監視記録の残し方

- 確認日時
- 確認者ロール
- 対象ページ
- desktop / mobile
- 結果: OK / NG
- 補足: private 記録への参照IDのみ

## 即時エスカレーション条件

- Home の初期表示が崩れる
- FAQ の開閉が壊れる
- Form の転送に失敗する
- Privacy の可読性が著しく落ちる
- 主要導線で 404 / 500 が発生する
- スマホ表示で業務上無視できない崩れがある
